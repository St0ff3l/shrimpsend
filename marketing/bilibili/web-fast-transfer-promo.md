## 一、纯口播文本（复制给配音 / TTS / 数字人）

```
你有没有过这种瞬间——

手机拍了一张图，想发到电脑继续修。
电脑下好了安装包，想丢给旁边的 Windows。
浏览器里复制了一段命令，想立刻出现在手机上。

然后你发现，你要么压缩，要么登录网盘，要么上传，复制链接，再打开链接，再下载。

明明设备就在旁边，文件却要绕地球一圈。

更糟的是公司电脑。

不能装软件，不能登私人网盘，U 盘可能被拦。
你只能发邮件给自己，或者把文件拆成好几份，一份一份传。

大文件传到一半断了——从头再来。

我们做了一个工具，叫虾传。

它的逻辑很简单：
不是生成链接再转发，
而是像给自己发消息一样——
选一台设备，内容直达。

手机、电脑、网页，都在同一个设备会话里。

网页版不用安装。
打开浏览器，登录，选目标设备，拖进去——开始传。

同在一个 Wi-Fi 下，
网页对网页、手机对网页，
走局域网直连，
不必先上传到云端，再下载回来。

这就是为什么它快。
不是魔法，是路径选对了——
能直连，就不绕公网。

文字、剪贴板、截图、视频、安装包、压缩包——
都可以在同一条会话里连续发送。

不用每次新建分享链接。
不用每次复制粘贴链接。
发完就在那台设备的会话里，随时再找得到。

临时用别人的电脑？
公司机器装不了客户端？
网页版就是为你准备的。

打开即用，传完就走。
你的文件，直达你的设备。

当然，不是所有网络都能直连。
跨网段、复杂 NAT 环境下，
虾传还有后备通道兜底——
先求快，再求稳。

如果你也受够了「上传、复制链接、下载」这套流程——

试试虾传网页版。
打开浏览器，选设备，拖文件。
一端发送，处处可达。

链接在简介。
评论区告诉我，你最想用它传什么。
```

## 六、注意事项（剪辑时请勿夸大）

- 断点续传：目前为原生客户端之间能力，客户端 ↔ 网页暂不支持
- 网页端局域网：同网段内可参与直连；需登录后加入设备会话
- 本地 mDNS 自动发现：仅原生客户端支持
