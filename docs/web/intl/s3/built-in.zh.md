# 内置 S3（国际版）

国际版 ShrimpSend 提供 **平台托管的对象存储**，无需填写 Endpoint、Bucket 或密钥即可在广域网场景下完成文件传输。

## 适用场景

- 你暂时不想配置自建对象存储。
- 设备跨网络，局域网 / WebRTC 直连失败，需要兜底通道。
- 文件仅用于传输中转，而非长期网盘备份。

## 工作原理

内置 S3 使用 **Cloudflare R2** 作为托管后端，默认桶名为 `shrimpsendfiles`。

- **无需配置**：在设置页保持「内置 S3」模式即可。
- **兜底通道**：系统优先尝试局域网 / WebRTC；直连不可用时再使用内置 S3。
- **按会员额度计量**：托管上传额度按会员档位和 UTC 自然月统计。
- **最多保存 30 天**：托管文件通常最多保留 30 天，之后可能由系统自动删除。
- **不是长期网盘**：仅用于传输中转，不承诺长期备份。

![设置页中的内置 S3 模式](/docs/s3/built-in/01-hosted-mode.png)

## 如何启用

1. 打开 **设置 → S3**。
2. 若当前为「未启用」或「自建 S3」，选择 **使用内置 S3**（或等价选项）。
3. 无需填写 Endpoint / 密钥；保存后即可在跨网场景下传输。

若你之后希望文件存放在自己控制的存储中，可切换为 [自建 S3 基本了解](/zh/docs/s3/overview) 并按左侧厂商指南配置。

## 与自建 S3 的区别

| 项目 | 内置 S3 | 自建 S3 |
| --- | --- | --- |
| 配置 | 无需 Endpoint / 密钥 | 需完整配置 + CORS |
| 数据位置 | 平台托管 R2 | 你的对象存储 |
| 额度 | 会员月度配额 | 由云厂商计费 |
| 保留 | 通常最多 30 天 | 由你自行管理 |

## 常见问题

**内置 S3 需要配置 CORS 吗？**  
不需要。CORS 由平台在托管桶上维护。

**如何查看本月用量？**  
在设置 → S3 的内置 S3 卡片中查看「本月已用」进度。

**能否与自建 S3 随时切换？**  
可以。若曾保存过自建配置，可在设置页切回；切回自建前请确认桶与 CORS 仍有效。
