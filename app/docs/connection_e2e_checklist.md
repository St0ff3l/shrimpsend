# 连接链路联调清单（P2P 重构后）

## 目标

验证以下闭环是否成立：

1. 默认自动选择最佳链路。
2. Android↔Android、iOS↔iOS 可通过平台直连建立会话。
3. Android↔iOS 不展示平台直连，继续走 HTTP / WebRTC / S3。
4. 手动切换后立即检测，并支持恢复自动模式。
5. 平台直连发送/接收文件时，聊天页进度和最终状态正确。

---

## 联调前准备

1. 两端均升级到同一提交版本。
2. 两端登录同一账号，并确保设备列表可见对端。
3. Android 端授权：
   - 附近设备权限
   - Wi-Fi/定位相关权限（按 ROM 提示）
4. iPhone 端授权：
   - 本地网络权限（Bonjour / WiFi Direct plugin）
5. 准备一个小文件（1-5MB）和一个大文件（>100MB）。

---

## 场景 1：Android ↔ Android（WiFi Direct）

### 步骤

1. A、B 两台 Android 进入彼此会话。
2. 观察顶部连接栏是否出现 `WiFi Direct` 方式。
3. 在“切换”弹窗的“平台专属操作”区点击“搜索”或“广播”。
4. 在另一端执行互补操作后点击“连接”。
5. 建链成功后发送小文件和大文件各一次。

### 预期

- 平台专属操作区显示为 `WiFi Direct`。
- 建链成功后顶部文案出现“已通过 WiFi Direct 连接 ...”。
- 文件发送过程中显示进度，发送完成后状态更新为已发送。
- 接收端能看到接收进度和最终文件消息。

---

## 场景 2：iOS ↔ iOS（Multipeer）

### 步骤

1. A、B 两台 iPhone 进入彼此会话。
2. 观察顶部连接栏是否出现 `Multipeer` 方式。
3. 在“切换”弹窗的“平台专属操作”区完成搜索/广播/连接。
4. 发送小文件和大文件各一次。
5. 手动切换到 HTTP 或 WebRTC，再点击“恢复自动”。

### 预期

- 平台专属操作区显示为 `Multipeer`。
- 平台直连成功后顶部连接栏显示已连接状态。
- Multipeer 不可用时可手动切回 HTTP / WebRTC。
- 恢复自动后重新回到编排器推荐结果。

---

## 场景 3：Android ↔ iOS（无平台直连）

### 步骤

1. Android 与 iPhone 进入彼此会话。
2. 观察顶部连接栏和“切换”弹窗中的方式列表。
3. 断开同网段条件，观察自动降级。
4. 分别通过 HTTP / WebRTC / S3 发送文件。

### 预期

- 不显示 `WiFi Direct` / `Multipeer` 平台直连方式。
- 自动选择只会在 `附近 / HTTP / WebRTC / S3` 之间切换。
- 平台直连操作区不会误导用户出现跨平台连接入口。

---

## 场景 4：全部直连失败 -> S3 兜底

### 子场景 A：S3 已配置

1. 人为让 LAN / WebRTC / 平台直连 都不可用。
2. 进入会话观察自动状态。
3. 发送文件。

**预期**
- 自动回退 `S3`，并可正常发送。

### 子场景 B：S3 未配置

1. 清空 S3 配置。
2. 进入会话并触发直连失败。
3. 点击 Smart Link 中“去配置 S3”。

**预期**
- 顶部出现“直连不可用，且 S3 未配置”提示。
- 点击按钮可跳转到 `/settings/s3`。

---

## 场景 5：手动切换与并发检测

### 步骤

1. A 与 B 同时进入同一会话，观察顶部状态与模式条。
2. A 端点击“切换”，选择 `HTTP` / `WebRTC` / `WiFi Direct` 或 `Multipeer`。
3. 在 A 端检测进行时，连续快速点击同一模式与刷新按钮。
4. B 端保持不操作，观察 B 的模式是否被 A 自动改变。

### 预期

- 每次切换都会先弹出线路明细弹框。
- 工具栏进入“检测中”后再更新为可用/不可用。
- 同一设备对的探测任务具备互斥与去重。
- A 的检测结果不会自动改写 B 的模式。

---

## 场景 6：统一连接栏与平台专属操作区

### 步骤

1. 进入任意会话，观察聊天区顶部连接栏。
2. 点击“切换”，查看弹窗中的方式列表和平台专属操作区。
3. 分别在 Android、iPhone、跨平台会话里观察差异。

### 预期

- 顶部仅有一个连接栏。
- 平台专属操作区在 Android 显示 `WiFi Direct`，在 iPhone 显示 `Multipeer`。
- Android↔iOS 会话中平台专属操作区不出现误导性直连入口。

## 关键日志关键字

建议过滤以下关键字定位问题：

- `PlatformP2pController.`
- `_showFileSendModal sendMode=`
- `probe.run source=`
- `chat_screen publication type=`

---

## 常见失败与排查

1. **Android 搜不到设备**
   - 检查 `Nearby devices`、定位权限、Wi-Fi 是否开启。
2. **iPhone 无法发现对端**
   - 检查本地网络权限与 Bonjour service 是否已授权。
3. **平台直连已建链但发送失败**
   - 先看顶部是否仍显示已连接，再看发送进度日志是否开始。
4. **Android↔iOS 误显示平台直连**
   - 检查 `platformP2p` 过滤条件是否只允许同平台移动端。
